4、cookie和session的区别

  • cookie是把用户的数据写给用户的浏览器，浏览器进行保存（可以保存多个）
  • session是把用户的数据写到用户独占的session中，服务器端保存（保存重要信息，减少服务器资源浪费）

5、使用场景

  • 保存用户登陆信息
  • 购物车信息
  • 在访问网站经常会使用的数据可以将其保存在Session中

6、JSP

JSP是Java Server Pages的缩写，它是一种动态网页开发技术，允许开发人员在HTML页面中嵌入Java代码。JSP页面在服务器上被解析成Java Servlet，然后由Servlet容器来处理请求并生成响应。

与传统的HTML页面相比，JSP可以使开发人员更容易地生成动态Web页面，因为它允许在HTML页面中嵌入Java代码，并且可以使用Java的各种功能来动态地生成页面内容。开发人员可以使用JSP编写Web应用程序的用户界面、数据访问逻辑、业务逻辑等。

JSP页面通常使用JSTL（JSP Standard Tag Library）和EL（Expression Language）来访问和处理JavaBean，这些都是Java Web应用程序的核心组件。JSP也可以与Java Servlet、JDBC（Java Database Connectivity）和其他Java技术一起使用，使得Javaweb应用程序的开发变得更加高效、简便。

4、内置对象

在 JSP 中，有九大内置对象，分别是：

  1. request 对象：代表客户端的请求，包含了客户端提交的所有请求参数和信息。例如：新闻
  2. response 对象：代表服务器的响应，包含了服务器返回给客户端的所有响应参数和信息。
  3. pageContext 对象：代表 JSP 页面的上下文，可以用于访问 JSP 页面的其他对象。
  4. session 对象：代表客户端的会话，可以用于存储和检索客户端的状态信息。例如：购物车
  5. application 对象：代表服务器的全局上下文，可以用于存储和检索服务器的状态信息。例如：聊天数据
  6. out 对象：代表服务器的输出流，可以用于向客户端发送响应。
  7. config 对象：代表 JSP 页面的配置信息，可以用于获取 JSP 页面的初始化参数。
  8. page 对象：代表当前 JSP 页面本身，可以用于获取 JSP 页面的信息和属性。
  9. exception 对象：代表 JSP 页面中发生的异常，可以用于捕获和处理 JSP 页面中的错误。
